# Intro
---
[Dreamhack][Dreamhack]의 WarGame 中 [tmitter][tmitter]문제에 대한 **Write-up**을 작성하고자 한다.

# Analysis
---
#### Site
사이트에 접근 시 로그인 페이지(Sign in)와 회원가입 페이지(Sign Up)가 존재하고, 로그인 시 자신만의 tmeet을 작성할 수 있는 페이지가 나타난다.
![site](/assets/tmitter/01_analysis_1.png){: style="border: 1px solid;"}

#### Query
아래의 쿼리를 참고하여 **admin**계정으로 로그인 시 FLAG 획득이 가능하다. 기존에 admin 계정이 데이터베이스에 저장되어 있기 때문에 이를 우회하여 회원가입을 수행해야 한다.
```sql
create table tmitter_user(
    idx int auto_increment primary key,
    id char(32),
    ps char(32)
);
```

#### char Type
저장 공간의 효율성과 국가별 언어 데이터 크기 등을 고려하여 확장성이 좋아 일상생활에서 자주 사용하는 **varchar/varchar2 Type**과, 문제에서 사용하는 **char** 타입의 차이점을 아래와 같다.
- char() : 고정 길이 문자열. 각 공간이 할당된 후 변수의 길이가 공간보다 작을 경우 그 차이 길이만큼 공백으로 채워진다.
- varchar/varchar2() : 가변 길이 문자열. 공간이 할당되지만 변수의 길이만큼 가변적으로 메모리에 공간을 할당하여 실제 데이터 길이만큼 저장 공간을 사용한다.

# Exploit
---
- `UNION SQL Injection` 이용
- `char Type` 특성 이용

#### UNION SQL Injection 이용
tmeet을 입력하는 페이지에 SQL Injection 공격에 대한 방지가 제대로 되어 있지 않다. tmeet 입력 후 저장 시 **INSERT 쿼리문**을 이용하여 데이터베이스에 저장한다. 따라서 **UNION SQL Injection**를 이용하여 admin계정의 패스워드를 tmeet으로 저장한다.
```plainText
',null),(null,'admin',(select ps from tmitter_user where id='admin'),null)#
```
![password](/assets/tmitter/02_exploit_1.png){: style="border: 1px solid;"}

획득한 admin 계정의 패스워드를 이용하여 로그인 시 FLAG 획득이 가능하다.
![FLAG](/assets/tmitter/02_exploit_2.png){: style="border: 1px solid;"}

#### char Type 특성 이용
회원가입 시 ID에 대한 문자열 길이 검증을 **클라이언트 단**에서 수행하며 데이터베이스에 **char Type**으로 저장되어 있다.

char Type 자료형은 데이터베이스에 설정한 공간을 벗어나는 데이터를 입력하는 경우, 정해진 문자 길이를 넘어가는 데이터를 삽입하면 취약점이 발생하는 경우가 생긴다.
예를 들어 char(10)으로 데이터 크기를 설정하면, `admin` `admin     a`를 동일 `admin`으로 인실할 수 있다.
```plainText
admin == admin     a
```

클라이언트 단 ID 문자열 길이 검증을 우회하기 위하여 maxlength를 33으로 변조하여 ID를 `admin___________________________a`, PW를 임의의 패스워드(test1234)로 설정하여 회원가입한다.
![client](/assets/tmitter/02_exploit_3.png){: style="border: 1px solid;"}

해당 사이트는 `char Type`을 사용함에도 데이터베이스에 저장 시 불필요한 공백을 제거하지 않거나, 입력된 문자열 길이를 검증하여 지정된 길이를 초과하는 경우 오류를 발생시키지 않는다.

따라서 `admin___________________________a` 계정으로 회원가입 시, 지정된 문자열 길이를 넘어서는 'a'는 삭제되어 데이터베이스 `admin___________________________`으로 저장된다. 이는 기존에 데이터베이스에 저장되어있던 `admin` 계정과 동일하게 처리되게 된다.

이렇게 생성한 admin 계정으로 로그인 수행 시, FLAG 확인이 가능하다.
![FLAG](/assets/tmitter/02_exploit_4.png){: style="border: 1px solid;"}
