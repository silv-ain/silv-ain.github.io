#### Code
아래와 같이 사이트에서 직접적으로 `/admin` 페이지에 접속 불가하도록 다양한 방법으로 막아놓았다. 따라서 주소를 입력받는 파라미터에 `SSRF 공격`을 수행해야 한다.

- IP가 미존재 또는 0.0.0.0 > [**Not a valid URL.**]
- 사설 IP 주소 > [**Can you access my admin page~?**]
- 응답값 내 Referer가 localhost 또는 127.0.0.1 > [**False**]
- 요청을 전송한 주소가 외부 > [**This is local page!**]

```Python
def check_get(url):
    ip = lookup(urlparse(url).netloc) # socket.gethostbyname(url)
    if ip == False or ip =='0.0.0.0':
        return "Not a valid URL."
    res=requests.get(url)
    if check_global(ip) == False: # (ipaddress.ip_address(ip)).is_global
        return "Can you access my admin page~?"
    for i in res.text.split('>'):
        if 'referer' in i:
            ref_host = urlparse(res.headers.get('refer')).netloc
            if ref_host == 'localhost':
                return False
            if ref_host == '127.0.0.1':
                return False
    res=requests.get(url)
    return res.text
```

# Exploit
---
- `URL Shortener`
- `Open Redirect` 공격
<br>

#### URL Shortener
`URL Shortener`는 URL을 축약해주는 서비스로, 서비스 주체가 되는 도메인의 하위에 축약된 URL에 원래의 URL로 Redirect하는 기능을 이용한다.
이용자는 서비스 주체가 되는 도메인의 URL을 거쳐 원래의 페이지로 이동하므로 공유 과정에서 다양한 정보(접속/통계 등) 획득이 가능하다.

아래와 같은 방법으로 임의의 `URL Shortener Site`를 이용하여 host 기반으로 검증하는 것을 우회한다.
```plainText
http://127.0.0.1:3333/admin > https://shorturl.at/P1vQb
```

#### Open Redirect 공격
`Open Redirect` 공격은 사용자의 입력을 기반으로 Redirect하는 기능을 이용하여, 공격자가 의도한 도메인으로 이동시키는 공격 방법이다.

`host 기반 필터링`을 수행할 때 다음과 같이 `@` 이전을 userinfo로, 이후를 host로 인식하는 점을 이용하여 브라우저에서 AT_SIGN 이후 도메인으로 Redirection 시킨다. 

이처럼 HTTP 프로토콜을 사용하는 임의의 사이트 URL을 작성한 후 실제로 접근하고자 하는 URL 주소를 작성하여 우회한다.
